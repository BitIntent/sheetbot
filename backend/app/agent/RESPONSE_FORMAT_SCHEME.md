# 大模型响应格式约束方案

## 目标
强制大模型按照前端渲染的格式回复，确保每个工具调用都给出实际应该输出的示例，用来约束大模型。

## 问题分析

### 当前问题
1. **响应格式不一致**：大模型可能返回各种格式的文本，前端渲染效果不一致
2. **工具调用输出不明确**：大模型不知道工具调用后应该如何描述结果
3. **缺少格式示例**：没有明确的格式模板指导大模型

### 前端消息类型
根据 `AIAssistant.jsx` 的分析，前端支持以下消息类型：
- `assistant` - 普通助手消息（Markdown 渲染）
- `error` - 错误消息（红色背景）
- `status` - 加载状态（蓝色背景，带动画）
- `thinking` - 思考状态（黄色背景）
- `tool_call` - 工具调用（蓝色背景，monospace）
- `tool_result` - 工具结果（绿色背景，monospace）
- `warning` - 警告（黄色背景）
- `backend_progress` - 后端进度（深蓝色背景，Markdown 渲染）

## 解决方案

### 方案1: 在 SYSTEM_PROMPT 中添加响应格式规范（推荐）

#### 1.1 添加响应格式章节
在 `SYSTEM_PROMPT` 中添加详细的响应格式规范：

```python
## 📋 响应格式规范（必须严格遵守）：

### 1. 操作执行前的说明
格式：
```
我将执行以下操作：
- [操作1描述]
- [操作2描述]
```

示例：
```
我将执行以下操作：
- 为"销售明细"工作表的第一行设置表头样式（加粗、背景色#217346、白色文字）
- 将成交单价、销售额(原价)、折扣额、销售额(净额)列设置为货币格式
```

### 2. 操作执行中的提示
格式：
```
🔧 正在执行：[操作描述]
```

示例：
```
🔧 正在执行：设置表头样式
🔧 正在执行：设置金额列格式
```

### 3. 操作完成后的总结
格式：
```
✅ **操作已完成！**

[操作1结果描述]
[操作2结果描述]

[可选：使用提示或后续建议]
```

示例：
```

我已成功为"销售明细"工作表的第一行设置了表头样式：
- ✅ **字体加粗**
- ✅ **背景色：#217346**（深绿色）
- ✅ **文字颜色：白色**

```

### 4. 错误情况
格式：
```
❌ **操作失败**

[错误原因]
[建议解决方案]
```

示例：
```

```

### 5. 查询类问题
格式：
```
根据当前数据，[查询结果]

**详细说明：**
[详细分析]
```

示例：
```
根据当前数据，共有 3 类产品。

**详细说明：**
- 产品A：出现 45 次
- 产品B：出现 32 次
- 产品C：出现 28 次
```

### 6. 需要用户补充信息
格式：
```
我注意到[问题描述]。让我确认一下：

[选项1描述]
[选项2描述]

请确认您希望选择哪个方案？
```

示例：
```

## 方案2: 在工具描述中添加输出示例

### 2.1 修改工具描述格式
在每个工具的描述中添加"输出示例"部分：

```python
@tool("set_range_style",
    """Apply styling to a range of cells.
    
    **输出格式示例（必须按照此格式回复）：**
    
    🔧 正在执行：设置表头样式
    
    ✅ **操作已完成！**
    
    我已成功为"[工作表名]"工作表的第[行号]行设置了样式：
    - ✅ **字体加粗**（如果设置了）
    - ✅ **背景色：[颜色代码]**（如果设置了）
    - ✅ **文字颜色：[颜色代码]**（如果设置了）
    
    样式已应用到[范围描述]，[效果描述]。
    
    **参数说明：**
    - sheet: 工作表名称
    - start_row, start_col, end_row, end_col: 范围坐标
    - style: 样式对象，包含 bold, backgroundColor, fontColor 等
    """,
    {...})
```

## 预期效果

1. **格式一致性**：所有回复都遵循统一格式
2. **用户体验**：用户能够清楚地知道操作是否成功，以及具体做了什么
3. **错误反馈**：错误信息格式统一，易于理解
4. **可维护性**：格式规范集中管理，易于更新

## 注意事项

1. **不要过度约束**：保持一定的灵活性，允许大模型根据具体情况调整描述
2. **示例要真实**：使用实际的数据结构（如"销售明细"、"销售额(净额)"等）
3. **保持简洁**：格式模板要简洁明了，不要过于复杂
4. **测试验证**：实施后需要测试各种场景，确保格式正确
